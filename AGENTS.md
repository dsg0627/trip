# AGENTS.md

## 目的

本文件用于说明在当前仓库中协作的智能体默认工作规范。

## 项目概览

- 项目名称：`java-demo`
- 主要语言：Java
- 构建工具：Maven

## 协作约定

- 保持修改范围尽量小且聚焦。
- 优先选择清晰、易维护的实现，而不是过度炫技的写法。
- 除非任务明确要求修改行为，否则应尽量保持现有行为不变。
- 当功能、配置或使用方式发生变化时，同步更新相关文档。

## 代码风格

- 遵循标准 Java 命名规范。
- 类和方法尽量保持单一职责。
- 避免引入不必要的依赖。
- 在项目规模较小时，优先保持简单的包结构。

## 验证要求

- 在本地工具可用时，运行相关测试或校验命令。
- 如果本地缺少工具或环境不完整，需要明确说明哪些内容尚未验证。

## 沟通要求

- 说明本次修改了什么内容。
- 说明已经执行了哪些验证。
- 清楚标注阻塞项、假设和后续建议。

## 后续可补充内容

- 架构说明
- 环境准备
- 测试策略
- 部署说明

## 编码说明

- 文件统一使用 UTF-8 编码保存。
- 如无特殊需要，优先使用普通中文标点和常见 Markdown 语法。
- 如果在 Windows 环境打开文件出现乱码，优先检查编辑器是否以 UTF-8 编码读取。
